# Project Charter Technical Summary

This summary captures the mandatory technical direction for **Process Transformation AI**.

## Product Vision

Process Transformation AI is a professional operational transformation platform. It is not a chatbot. It will support process understanding, As-Is construction, VSM, Lean waste detection, TOC restriction detection, automation opportunities, AI opportunities, disruptive opportunities, To-Be design, business case generation, and roadmap generation.

## Mandatory Principles

- Modular architecture.
- Clean separation between frontend, backend, data, documents, and tests.
- SOLID and clean code.
- No functionality outside the approved Product Backlog.
- Every delivery must be executable from Visual Studio Code.
- No backend, Google Sheets, Drive, or AI integration without an approved PB.
- Configuration and constants must be centralized.

## Technical Stack

- Frontend: HTML, CSS, JavaScript.
- Backend: Google Apps Script.
- Database: Google Sheets.
- Documents: Google Drive.
- Code repository: GitHub.
- IDE: Visual Studio Code.
